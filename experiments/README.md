# Experimentos

A pasta `experiments/` será usada para experimentos práticos e reproduzíveis sobre avaliação de sistemas com IA.

Nenhum resultado deve ser apresentado sem código, dados permitidos, metodologia, limitações e instruções suficientes para reprodução. Quando um experimento ainda estiver em planejamento, ele deve ser descrito como proposta ou roteiro.

## Organização

- `fairness/`: avaliações de vieses, métricas e impactos entre grupos.
- `explainability/`: explicações locais, globais e comunicação de limitações.
- `red-teaming/`: testes adversariais e avaliação de segurança.
- `avaliacao-de-risco/`: métodos para classificar, priorizar e mitigar riscos.

## Critérios mínimos

Um experimento futuro deve conter objetivo, hipótese, dados permitidos, método, métricas, limitações, instruções de execução e interpretação responsável dos resultados.
